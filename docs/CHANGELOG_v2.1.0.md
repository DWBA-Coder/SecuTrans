# SecuTrans v2.1.0 更新日志

## 🎯 主要更新

### 1. 新增现代加密算法支持

#### ✨ 安全算法（推荐使用）
- **AES-128/192/256**: 分别支持三种密钥长度，更灵活的配置
- **ChaCha20**: 现代流加密算法，高性能，安全性高
- **SM4**: 中国国家标准的分组密码算法
- **Camellia**: 日本开发的分组密码算法，与AES安全性相当

#### ⚠️ 传统算法（已标记为不安全，仅作演示）
- **DES**: 数据加密标准，56位密钥，已被破解
- **3DES**: 三重DES，速度慢，安全性不足
- **Blowfish**: 对称块加密，存在安全漏洞
- **RC4**: 流加密算法，存在严重安全缺陷

### 2. 新增GCM认证加密模式
- **AES-GCM**: 提供机密性和完整性保护的认证加密模式
- **安全优势**: 防止数据篡改，无需额外的完整性验证
- **性能优势**: 并行加密，适合现代CPU

### 3. 完整的菜单系统
- **文件菜单**: 选择文件、生成密钥对、退出
- **工具菜单**: 清空表单、清空日志、保存日志
- **帮助菜单**: 使用说明、关于信息
- **快捷键支持**: 
  - `Ctrl+O`: 选择发送文件
  - `Ctrl+K`: 生成测试密钥对
  - `Ctrl+R`: 清空所有表单
  - `Ctrl+L`: 清空日志
  - `Ctrl+S`: 保存日志
  - `Ctrl+A`: 关于
  - `F1`: 帮助

### 4. 安全标记系统
- **算法安全状态**: 实时显示算法是否安全
- **模式安全提示**: 标记不安全的工作模式
- **颜色编码**: 
  - 绿色 ✓: 安全，推荐使用
  - 红色 ⚠: 不安全，仅作演示

### 5. 界面优化
- **文件信息显示**: 选择文件时显示文件大小
- **更好的文件类型过滤**: 按类别组织文件类型
- **优化的布局**: 更直观的用户界面
- **实时状态反馈**: 更清晰的操作提示

## 🔧 技术改进

### 代码重构
- **模块化加密实现**: 更清晰的算法组织
- **错误处理优化**: 更友好的错误提示
- **配置文件更新**: 支持新算法和模式的配置

### 安全增强
- **完整性保护**: GCM模式提供内置完整性验证
- **密钥管理**: 更安全的密钥生成和存储
- **算法警告**: 明确标识不安全的加密选项

## 📋 使用建议

### 推荐配置
1. **算法选择**: AES-256 + GCM（最佳安全性）
2. **备选方案**: ChaCha20（高性能场景）
3. **国家标准**: SM4（符合国内标准要求）

### 安全提醒
1. **避免使用**: DES、3DES、Blowfish、RC4等已不安全的算法
2. **模式选择**: 避免使用ECB模式（存在模式泄露风险）
3. **密钥安全**: 妥善保管私钥文件，避免泄露

### 性能优化
1. **大文件传输**: ChaCha20在大文件场景下表现更好
2. **CPU优化**: AES-GCM在现代CPU上有硬件加速支持
3. **内存使用**: GCM模式内存占用相对较高

## 🐛 问题修复

### 修复的问题
- **算法选择器**: 更新后支持新算法的显示和选择
- **加密兼容性**: 确保新旧版本的文件传输兼容性
- **界面响应**: 优化了界面操作的响应速度

### 已知限制
- **SM4实现**: 当前使用AES作为替代实现，需要专门的SM4库
- **ChaCha20**: 需要较新版本的pycryptodome库支持

## 📦 依赖更新

### 新增依赖
- 无新增依赖，所有功能基于现有的pycryptodome库

### 版本要求
- **pycryptodome**: >= 3.19.0（支持ChaCha20和Camellia）
- **cryptography**: >= 41.0.7（保持现有版本）
